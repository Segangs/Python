## 가상화

- (의존성) 의존성 충돌을 많이 피할수 있다. - 프로젝트마다 (파이썬)라이브러리 관리. 바이너리 파일도 독립적으로 관리  
    커널까지 포함해서 실행. WSL은 커널을 제공한다.

- VM - 머신 수준에서 독립적인 환경을 제공한다 (GCP, 클라우드 등..)

- 도커 : WSL수준의 커널 제공.

- 환경 변수 설정 중요하다.



## 왜 임베디드 개발자에게 파이썬이 필요한가?

- 자동화 (배치)

- 데이터 분석 (시각화 / 통계)

- 대시보드, 서버, 클라우드 연동

- AI / 머신러닝 (모델 학습/ 예측/ 지능화)



## 파이썬이란

- 인터프리터 언어

- 문법이 간결 (가독성 높음)

- 동적 타이핑 (int같은 변수 타입을 미리 선언하지 않음. 런타임에서 변한다) ↔ 정적 타이핑 (C언어)  
    파이썬에서는 primative type가 없다. type → class + heap 메모리 사용

- 객체지향 지원 (클래스, 객체 기반 설계)

- 라이브러리 풍부
